# Vue Skill

## When to use

Use this skill for Vue components, frontend state, API calls, responsive UI, filters, and tables.

## Rules

- Reuse existing component and store patterns.
- Keep API errors visible and actionable.
- Preserve responsive behavior and table/filter state.
- Avoid large visual redesigns unless requested.
- Test component state and integration paths.

## Verification

Run the project's frontend lint, typecheck, and test commands when available.
